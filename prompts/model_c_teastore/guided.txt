Robustness testing background:
In robustness testing, the goal is to characterize system behavior in the presence
of erroneous input conditions. Tests are generated by applying systematic mutation
rules to each input parameter based on its data type. The following mutation rules,
derived from established robustness testing research, define the categories of
invalid inputs that should be applied to each parameter:

String parameter mutations:
  - Replace by null value
  - Replace by empty string
  - Replace by a predefined string (a fixed known string unrelated to the parameter)
  - Replace by string with nonprintable characters
  - Add nonprintable characters to the string
  - Replace by alphanumeric string (where a specific format is expected)
  - Add characters to overflow the maximum expected size

Number parameter mutations:
  - Replace by null value
  - Replace by empty value (omit the parameter)
  - Replace by -1
  - Replace by 1
  - Replace by 0
  - Add one to a valid value
  - Subtract one from a valid value
  - Replace by the maximum number valid for the type (e.g., Integer.MAX_VALUE)
  - Replace by the minimum number valid for the type (e.g., Integer.MIN_VALUE)
  - Replace by the maximum valid value plus one (overflow)
  - Replace by the minimum valid value minus one (underflow)

endpoints listed above. Each endpoint parameter has a data type (string or integer).
For each parameter, select the most distinct mutations from the applicable rules.
Prioritise mutations that are structurally different from each other — for example,
"replace by null" and "replace by empty string" are different mutations even though
both represent missing data.
