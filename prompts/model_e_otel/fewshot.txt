Robustness oracle:
The system passes a robustness test if the HTTP response status code is less than 500.
A 2xx, 3xx, or 4xx response means the system handled the input gracefully.
A 5xx response is a robustness failure.

Your test class must extend OTelShopBaseTest which provides:
  HttpResponse<String> get(String path) throws Exception
  HttpResponse<String> postJson(String path, String jsonBody) throws Exception
  HttpResponse<String> delete(String path) throws Exception
  void assertNoServerError(HttpResponse<?> response)

The postJson() method automatically sets Content-Type: application/json.

Examples of correctly written robustness tests:

@Test
public void test_R1_ProductNonExistentId() throws Exception {
    HttpResponse<String> response = get("/api/products/DOESNOTEXIST999");
    assertNoServerError(response);
}

@Test
public void test_R1_CartEmptySessionId() throws Exception {
    HttpResponse<String> response = get("/api/cart?sessionId=");
    assertNoServerError(response);
}

@Test

    assertNoServerError(response);
}

@Test

    String body = "{\"from\":{\"currencyCode\":\"ZZZ\",\"units\":10,\"nanos\":0},\"toCode\":\"USD\"}";
